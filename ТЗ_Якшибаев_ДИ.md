**1.4. Область применения**

Система предназначена для использования участниками чемпионата по программированию IT-ВыСотка (школьники 8-11 классов), волонтерами для контроля присутствия участников в день проведения мероприятия.

## 2. ЦЕЛИ И ЗАДАЧИ СОЗДАНИЯ СИСТЕМЫ

**2.1. Цель разработки**

Создание пользовательской части веб-приложения и панели волонтера для обеспечения удобной регистрации участников чемпионата, управления командами, доступа к информации о мероприятии, а также контроля присутствия участников.

**2.2. Задачи разработки**

1. Разработка системы регистрации и авторизации пользователей
2. Реализация личного кабинета участника с возможностью редактирования профиля
3. Создание системы управления командами с пригласительными ссылками
4. Разработка публичных страниц сайта с информацией о чемпионате
5. Реализация системы восстановления пароля
6. Создание компонентов пользовательского интерфейса
7. Разработка системы email-уведомлений для участников
8. Реализация панели волонтера для контроля присутствия участников
9. Создание функционала редактирования данных участников волонтерами
10. Обеспечение безопасности пользовательских данных

## 3. ТРЕБОВАНИЯ К СИСТЕМЕ

**3.1. Требования к функциональным характеристикам**

**3.1.1. Система регистрации и авторизации**

Функция: Регистрация новых пользователей и вход в систему

Требования: регистрация с указанием email, пароля, ФИО, класса (8, 9, 10, 11), школы (каскадный выбор: регион → город → школа), валидация всех полей формы регистрации, проверка уникальности email, хеширование паролей, отправка письма активации на email, активация аккаунта по ссылке из письма, авторизация по email и паролю, выдача JWT токенов (access и refresh), автоматическое обновление токенов, выход из системы с удалением токенов.

**3.1.2. Восстановление пароля**

Функция: Сброс забытого пароля

Требования: запрос на сброс пароля по email, отправка письма со ссылкой для сброса пароля, проверка токена сброса пароля, установка нового пароля, валидация нового пароля (минимум 6 символов), хеширование нового пароля.

**3.1.3. Личный кабинет участника**

Функция: Управление профилем и просмотр информации об участии

Требования: просмотр данных профиля (ФИО, email, класс, школа, формат участия, команда), редактирование профиля (изменение ФИО, класса, школы), изменение формата участия (индивидуальное/командное), добавление ссылки на эссе, просмотр информации о команде (если участник в команде), просмотр присвоенного места (если есть), скачивание сертификата (если выдан), отображение статуса регистрации на чемпионат.

**3.1.4. Система управления командами**

Функция: Создание команд и управление составом

Требования: создание команды с указанием названия (уникальное, 3-50 символов), автоматическое назначение создателя капитаном команды, генерация уникальной пригласительной ссылки для команды, присоединение к команде по пригласительной ссылке (до 3 участников), автоматическое изменение формата участия на "командное" при вступлении в команду, просмотр списка участников команды, исключение участника из команды (только капитан), выход из команды, удаление команды (только капитан), автоматическое удаление команды при выходе последнего участника.

**3.1.5. Работа со школами**

Функция: Выбор образовательного учреждения

Требования: каскадный выбор школы (сначала регион, затем город, затем школа), загрузка списка регионов из базы данных, загрузка списка городов выбранного региона, загрузка списка школ выбранного города, поиск по названию школы, импорт базы данных школ России из CSV файла (более 40000 записей).

**3.1.6. Публичные страницы сайта**

Функция: Информирование посетителей о чемпионате

Требования: главная страница с описанием чемпионата и призывом к регистрации, страница "О чемпионате" с подробной информацией о мероприятии, страница "Регламенты" с документами и правилами проведения, страница "Галерея" с фотографиями с прошлых чемпионатов, страница "Результаты" с возможностью просмотра результатов по годам, страница "Контакты" с информацией для связи с организаторами, страница согласия на обработку персональных данных для родителей, страница согласия на обработку персональных данных для участников, адаптивный дизайн для мобильных устройств и планшетов.

**3.1.7. Email-уведомления**

Функция: Отправка уведомлений участникам

Требования: отправка письма активации аккаунта с уникальной ссылкой, отправка письма для сброса пароля с временной ссылкой, отправка уведомления о выдаче сертификата со ссылкой для скачивания, отправка уведомления о победе (1-3 места) с поздравлением, использование SMTP для отправки писем, корректное отображение писем в различных почтовых клиентах.

**3.1.8. Панель волонтера**

Функция: Контроль присутствия участников на чемпионате

Требования: авторизация волонтера с отдельными учетными данными, главная панель с общей статистикой присутствия (общее количество участников, присутствующие, отсутствующие, процент явки), просмотр списка участников с рассадкой по аудиториям, фильтрация участников по аудиториям, поиск участников по ФИО, школе, команде, отметка присутствия отдельного участника (кнопка "Присутствует"/"Отсутствует"), массовая отметка присутствия для всей аудитории, отображение времени отметки присутствия, история отметок присутствия для каждого участника, статистика по аудиториям (количество присутствующих/отсутствующих в каждой аудитории), экспорт списков участников по аудиториям в PDF.

**3.1.9. Редактирование участников волонтером**

Функция: Корректировка данных участников в день чемпионата

Требования: редактирование ФИО участника, редактирование класса участника, редактирование школы участника, сохранение истории изменений профиля (кто, когда, что изменил), просмотр истории изменений профиля участника, валидация изменяемых данных.

**3.1.10. Компоненты интерфейса**

Функция: Переиспользуемые элементы пользовательского интерфейса

Требования: компонент навигации (Navbar) с меню и кнопками входа/регистрации, компонент футера (Footer) с контактной информацией и ссылками, компонент уведомлений (Toast) для отображения сообщений об успехе/ошибке, компонент модальных окон (InfoModal) для отображения информации, компонент диалогов подтверждения (ConfirmDialog) для критических действий, компонент редактирования профиля (EditProfileModal), компонент выпадающего списка с поиском (SearchableSelect) для выбора школы, компонент секции спонсоров (SponsorsSection) для главной страницы, компонент секции задач (TasksSection) для отображения этапов чемпионата, компонент публичной галереи (PublicGallery) для отображения фотографий.

**3.2. Требования к надежности**

Система должна обеспечивать стабильную работу в период регистрации участников. Все пользовательские данные должны быть защищены от несанкционированного доступа. Пароли должны храниться в хешированном виде. Токены аутентификации должны иметь ограниченный срок действия. Система должна корректно обрабатывать ошибки и выводить понятные сообщения пользователю. Валидация всех входных данных на стороне клиента и сервера.

**3.3. Требования к составу и параметрам технических средств**

Минимальные требования к клиенту (устройство участника): процессор 1 ядро 1.0 GHz, оперативная память 2 GB RAM, разрешение экрана 360x640 (мобильные устройства), веб-браузер Google Chrome 90+, Mozilla Firefox 90+, Safari 14+, Microsoft Edge 90+.

Рекомендуемые требования к клиенту: процессор 2 ядра 1.5 GHz, оперативная память 4 GB RAM, разрешение экрана 1366x768 или выше, веб-браузер последняя версия Google Chrome, Mozilla Firefox или Safari.

**3.4. Требования к информационной и программной совместимости**

Backend: Node.js 20.0.0 или выше, Express.js 5.1.0, PostgreSQL 15.0 или выше, Sequelize ORM 6.35.0, JWT (jsonwebtoken 9.0.2), Nodemailer 6.9.0, bcrypt 5.1.1, uuid 11.0.3.

Frontend: React 19.2.0, MobX 6.15.0, React Router 7.1.3, Axios 1.7.9.

Инфраструктура: Docker 20.10+, Docker Compose 2.0+, Nginx 1.25+ (Alpine).

Форматы данных: импорт школ CSV (UTF-8), сертификаты PDF, экспорт списков участников PDF.

**3.5. Требования к методам и средствам защиты информации**

Использование JWT токенов для аутентификации. Access токен: срок действия 15 минут. Refresh токен: срок действия 30 дней. Хеширование паролей с использованием bcrypt (10 раундов). Защита маршрутов с помощью middleware аутентификации. Валидация всех входных данных на стороне клиента и сервера. Защита от SQL-инъекций через использование ORM Sequelize. Защита от XSS-атак через экранирование данных. CORS настроен для работы только с доверенными доменами. HTTP-only cookies для хранения refresh токенов. Автоматический выход при истечении срока действия токенов.

**3.6. Требования к эргономике и технической эстетике**

Современный и привлекательный дизайн пользовательского интерфейса. Адаптивный дизайн для работы на мобильных устройствах, планшетах и десктопах. Единый стиль оформления всех страниц. Интуитивно понятная навигация. Понятные подписи к полям форм. Всплывающие подсказки для сложных элементов. Быстрая обратная связь на действия пользователя (уведомления, индикаторы загрузки). Понятные сообщения об ошибках с указанием способа исправления. Доступность для пользователей с ограниченными возможностями (контрастность, размер шрифтов).

Кафедра Вычислительной техники и инженерной кибернетики Уфимского государственного нефтяного технического университета, именуемая в дальнейшем «Заказчик», в лице заведующего кафедрой Зарипова Д.М., с одной стороны, и студент Уфимского государственного нефтяного технического университета группы БПО-22-02 Якшибаев Данир Ильдарович, именуемый в дальнейшем «Исполнитель», с другой стороны, составили настоящий акт о нижеследующем.

Исполнителем выполнена, а Заказчиком принята разработка пользовательской части веб-приложения и панели волонтера для проведения чемпионата по программированию IT-ВыСотка согласно утвержденному Заказчиком техническому заданию от 20 ноября 2025 года. Разработка включает в себя систему регистрации и авторизации участников, личный кабинет участника с возможностью редактирования профиля, систему создания и управления командами, публичные страницы сайта с информацией о чемпионате, панель волонтера для контроля присутствия участников с возможностью редактирования данных участников и просмотра статистики. Система развернута на сервере по адресу http://it-vysotka.rusoil.net и готова к использованию. Работа выполнена в период с 01 ноября 2025 года по 01 февраля 2026 года полностью и в срок. Заказчик по качеству и срокам претензий не имеет.

Настоящий акт внедрения составлен в двух экземплярах, имеющих одинаковую юридическую силу, по одному экземпляру для каждой из Сторон.

Разработанная система передается для использования организаторами и участниками чемпионата по программированию IT-ВыСотка кафедры ВТИК УГНТУ.
